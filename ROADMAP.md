# Tinker Studio Roadmap

## Near-Term Priorities

1. Bring user experience, interface quality, general functionality, and type/layout polish to a professional standard.
2. Add trace corpus support with explicit selection of which corpus parts go into a run. Traces should be a curated family for tools, reasoning, and workflow records, not random local Codex session import.
3. Generalize the app beyond a Tinker-only workflow so the corpus, run, and evaluation interfaces can support other training or evaluation backends.

## Corpus Families

- `posts`: social/media rows with reply structure and metadata preserved.
- `writing`: markdown documents and other authored text with variable user tags.
- `traces`: future curated workflow/tool/reasoning records with explicit inclusion controls.
